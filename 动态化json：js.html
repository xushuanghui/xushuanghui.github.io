会员动态化json/js

这一套思想实现的技术点都是js和原生交互，把通用的、基础的功能抽成原生组件，而动作、传值(组件之间、界面之间)通过js来完成

// "delegate","realHide","subComponentModels","parentComponent","valChangeSubject"

{
    "pageTitle": "设置会员体系",
    "pageId": "页面id",
    "script": "member_system_step_one.js", // 当前模板页对应的js上下文
    "onload": "onload()", // js上下文中的onload()方法
    "onChange": "",
    "leftBarConfig": {
        "title": "返回",
        "action": "onLeftBtnClicked()"
    },
    "rightBarConfig": {
        "title": "下一步",
        "action": "onRightBtnClicked()"
    },
    "header": {
        "components": []
    },
    "components": [
      {
        "component": "组件名称(唯一性)",
        "config": {
          "id": "s_0_4(具体模板中的id)",
          "name": "对应上传/下发时的key",
          "value": "即当前组件上传/下发的值",
          "subpage": "子页面",
          "subComponents": [子组件],
          "parentComponent"："父组件(原生代码中使用)",
          "preValue": "初始值",
          "prop": {  
            "title": "▪︎ 结束日期",
            "placeholder": "必选",
            "requiredTip": "请设置结束日期项"
          },
          "style": { //样式
            "descTextColor": "0xff0000"
          },
          "realHide": "组件隐藏",
        }
      }
    ],
    "footer": {
        "components": []
    },
}

 

解析

jsonModel -> item -> view

模板页 解析成原生对象(此时components仍是Array) 加载"script"指定的js文件

由components生成对应的组件model、item，给models设置value(有的化)、尝试设置组件预览状态、通知各model数据准备完成、尝试执行组件的onChange/onload方法(同时存在只执行onload)、执行整个页面的onload方法、reload页面

2、js调用原生

JSExecutor.log()  // log
JSExecutor.alertMsg()、JSExecutor.alertMsgTitlesAction() // 弹窗提示
JSExecutor.getVal()、JSExecutor.setVal() 	// 获取/设置组件的值 
JSExecutor.getProp()、JSExecutor.setProp()   // 获取/设置prop的属性
JSExecutor.getExtendInfo // 获取聚合数据
JSExecutor.getFormDataCallBack // 获取所有组件的value
JSExecutor.requestWithAction() // 提交数据
JSExecutor.routeWithUrl()、JSExecutor.routeWithUrlCallBack() // 路由跳转
JSExecutor.removePage()、JSExecutor.exit() // 活动页返回/退出
JSExecutor.gotoPage() // 跳转到下个json配置页，释放之前的页面
JSExecutor.gotoSubpage() // 跳转到下个json配置页
JSExecuter.changeLeftBarConfig()、JSExecutor.changeRightBarConfig() // 导航按钮
JSExecuter.changePageTitle() // 页面标题
JSExecuter.execFuncWithParamsInPage()  // 执行前一个页面的js方法
